# 面试对齐 (Interview Alignment)

## 动机
在准备一场新面试之初使用此技能。它的作用绝对不是开始模拟提问，而是纯粹用于发现能力断层，并通过苏格拉底式的引导，确立面试中的（进攻和防御）总体战术策略。

## 1. 强制信息摄入 (Hard-Gate)
你必须强制要求用户提供以下输入（如果尚未提供）：
1. **JD (职位描述)** 或目标职位要求。
2. 候选人的 **简历 (Resume)**。

**严禁** 在这两项材料提供完毕之前推进下一步。

## 2. 差距分析与苏格拉底式策略引导
在后台默默对比 JD 和简历之间的错位情况，找出 2-3 个核心的高危风险区（例如：JD 重度要求 Go 语言，而简历上只有 Java 数据）。

将你发现的差距点告知用户，随之使用**苏格拉底发问法**逼迫用户建立应对策略。
*示例：* “你缺乏分布式系统的实战经验，而这在 JD 里被高频提及。具体来说，当被问到扩展性问题时，你打算如何将话题转移到你最擅长的数据库优化领域上去？”

**严禁** 直接提供解决方案。逼迫用户去定义他们自己的防守打法。通过不断迭代，直到确立清晰的有攻有守的面试策略。

## 3. 生成工件与接力 (Terminal State)
一旦双方敲定好整体战术策略，你必须生成两个文件来进行状态持久化封存：

1. **建立简历快照 (Resume Snapshot)**
**保存要求：** `docs/interviews/YYYY-MM-DD-[Company]-[Role]/resume-snapshot.md`
必须将用户最初提交的那份简历原文一字不落地保存在这，作为历史快照，供日后实战复盘时进行审查。

2. **战术规格书 (Strategy Document)**
**保存要求：** `docs/interviews/YYYY-MM-DD-[Company]-[Role]/strategy.md`
(请使用实际的日期、公司名和角色名创建独立的实体文件夹，例如阿里、腾讯)。

该文档必须包含以下板块：
- 基础背景要求 (Context)
- 断层与差距分析 (Gap Analysis)
- 进攻战术：要重点突出的长板 (Offensive Strategy)
- 防御战术：遇到短板的扬长避短机制 (Defensive Strategy)

**当此文档保存完毕后，声明并调用下一个接力技能：**
> "策略已定义并保存为您专属的战略文档。请审查该文档，如果确认无误，我们可以推进到下一阶段。我现在正在唤起 `@[/interview-planning]` 帮您生成训练计划。"
